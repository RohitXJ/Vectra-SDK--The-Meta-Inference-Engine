# Unknown Category and Live Evaluation Integration

This plan outlines the changes required to add an "Unknown" classification category for out-of-distribution predictions, and a "Live Evaluation" section to test the model dynamically on the final results page.

## User Review Required

> [!IMPORTANT]
> The threshold to classify an image as "Unknown" is highly dependent on the representations learned by the backbone. I propose setting the `unknown_threshold` to `1.5 * max(distance of support images to their true prototypes)`. This heuristic works well to allow some variance for query images while rejecting major outliers.

## Core Functionality

### 1. `VectraInference` Class
- **Initialization:** Load `.pt` model, initialize backbone, and load prototypes/thresholds.
- **Predict Method:**
    - Support for file paths (single or list).
    - Support for `PIL.Image` objects.
    - Support for `numpy` arrays (OpenCV frames).
    - Support for confidence scores (cosine similarity or distance).
    - Automatic handling of "Unknown" category rejection.

### 2. Live Stream Support
- Create high-level utilities to wrap OpenCV `VideoCapture`.
- Provide a simple API to run inference on a live camera stream with overlaid labels.

### 3. Packaging & Branding
- Standard `pyproject.toml` and `setup.py` for PyPI publishing.
- Documentation oriented towards developers wanting to integrate Vectra models into their own apps.
